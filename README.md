# MagicStitch — простой сайт с публикацией в GitHub

Этот проект может сначала казаться запутанным. Это нормально.

Здесь есть две важные вещи:

* сайт (то, что ты видишь)
* сервер (то, что делает публикацию в GitHub)

Если что-то не работает — почти всегда проблема не в коде, а в том, **как ты его используешь**. Этот README как раз про это.

---

# 🧩 Как устроен проект (без путаницы)

Проект состоит из трёх частей:

### 1. Сайт (frontend)

* `index.html`
* `admin.html`
* `main.js`

Это интерфейс, через который ты:

* добавляешь работы
* настраиваешь GitHub
* нажимаешь "опубликовать"

---

### 2. Сервер (backend)

* `server.py`

Он делает ровно одну вещь:

> принимает команды от сайта и отправляет файлы в GitHub

---

### 3. Конфиги (самое важное)

#### `config.json`

Публичные настройки:

```json
{
  "github": {
    "owner": "username",
    "repo": "repo-name",
    "branch": "main"
  }
}
```

#### `secrets.json`

Секреты (НЕ публикуются):

```json
{
  "github_token": "ghp_..."
}
```

---

# 🔑 Где вводить GitHub токен

Важно:
токен **больше не хранится в config.json**

Если ты его туда вставишь — он будет удалён.

---

## Правильный способ:

1. Открой файл:

```
secrets.json
```

2. Вставь:

```json
{
  "github_token": "ghp_ТВОЙ_ТОКЕН"
}
```

3. Сохрани файл

---

Если после этого не работает — проблема не в токене, а в том, что сервер его не читает.

---

# 🚀 Как запустить проект

### 1. Установи Python

Проверь:

```
python --version
```

---

### 2. Запусти сервер

```
python server.py
```

Ты увидишь:

```
Server started at http://localhost:8000
```

---

### 3. Открой сайт

❗ Важно:

Открывать нужно так:

```
http://localhost:8000/admin.html
```

НЕ так:

```
file:///...
```

---

# 📤 Как работает публикация

Когда ты нажимаешь "Publish":

1. Сайт отправляет запрос → `/git-push`
2. Сервер:

   * читает `config.json`
   * читает `secrets.json`
   * собирает файлы
   * отправляет их в GitHub

---

# ❌ Частые ошибки (и почему они происходят)

## Ошибка: `Unknown endpoint`

Причина:

* запрос отправлен не туда

Решение:

* должен быть путь `/git-push`

---

## Ошибка: ничего не происходит

Проверь:

* запущен ли `server.py`
* открыт ли сайт через `localhost`

---

## Ошибка: GitHub не принимает

Проверь:

* есть ли `github_token` в `secrets.json`
* есть ли доступ к репозиторию

---
