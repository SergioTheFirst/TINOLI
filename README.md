# Broshio — сайт-галерея мастера украшений

> Если ты читаешь это — скорее всего, ты либо сам мастер, либо помогаешь кому-то запустить свой сайт. Оба варианта хороши. Здесь всё устроено так, чтобы не нужно было быть программистом.

**Живой сайт:** [broshio.ru](https://broshio.ru)

---

## Что это такое и зачем оно нужно

Это готовый сайт-галерея для мастера украшений ручной работы. Показывает работы, рассказывает о мастере, даёт посетителю способ написать.

Никаких баз данных. Никакого хостинга с настройкой сервера. Никаких `npm install` и прочего программистского колдовства. Только файлы, Python и GitHub.

**Как это работает в двух словах:**
- Фотографии и описания хранятся в папке `works/`
- Все тексты сайта — в одном файле `config.json`
- Локальный сервер (`server.py`) запускается одной командой
- Кнопка «Опубликовать» в админке сама отправляет всё на GitHub → сайт обновляется автоматически

---

## Быстрый старт

```bash
python server.py
```

Открой браузер: [http://localhost:8000](http://localhost:8000)

Чтобы добавить работу или изменить тексты — перейди на [http://localhost:8000/admin.html](http://localhost:8000/admin.html)

> **Требования:** Python 3.8 или новее. Работает на Windows (включая 32-bit), macOS, Linux. Никаких дополнительных библиотек устанавливать не нужно — только стандартный Python.

## Структура проекта

```
broshio/
├── index.html          # Главная страница
├── gallery.html        # Галерея всех работ
├── about.html          # О мастере
├── contact.html        # Контакты
├── admin.html          # Панель управления (только локально!)
├── server.py           # Локальный сервер
├── config.json         # Все тексты и контакты — редактируй здесь
├── secrets.json        # Токен GitHub — не коммитится, только у тебя
├── css/style.css       # Стили сайта
├── js/main.js          # Логика галереи, лайтбокс, анимации
└── works/
    ├── works.json      # Список всех работ с описаниями
    └── images/         # Фотографии украшений
```

`admin.html` и `server.py` — только для локальной работы. На хостинг они не нужны и не попадают в индекс поисковиков.

## Как добавить новую работу

1. Запусти `python server.py`
2. Открой [http://localhost:8000/admin.html](http://localhost:8000/admin.html)
3. Загрузи фото, заполни название и описание
4. Нажми **«Опубликовать»** — работа появится на живом сайте

Если хочешь загрузить два фото на одну работу (основное + при наведении) — оба поля есть в форме.

---

## Как изменить тексты

Открой `config.json` в любом текстовом редакторе (Блокнот, VS Code, Notepad++).

Например, чтобы сменить контакты:

```json
"contact": {
  "phone": "+7 (999) 123-45-67",
  "telegram": "@broshio",
  "instagram": "@broshio",
  "email": "hello@broshio.ru"
}
```

Сохранил файл → обновил страницу в браузере → изменения применились сразу.

## Публикация на GitHub

Чтобы сайт был доступен в интернете, нужно один раз настроить GitHub-токен:

1. Перейди в [github.com/settings/tokens](https://github.com/settings/tokens) → создай токен с правом `repo`
2. Открой [http://localhost:8000/admin.html](http://localhost:8000/admin.html) → вкладка **«Настройки»** → вставь токен
3. Нажми **«Опубликовать»** — сайт обновится на GitHub, Cloudflare Pages автоматически подхватит изменения

Токен сохраняется только локально в `secrets.json` — он никогда не попадёт в репозиторий.

## Деплой и хостинг

Сайт настроен для работы с **Cloudflare Pages**:
- Подключи репозиторий в [pages.cloudflare.com](https://pages.cloudflare.com)
- При каждом пуше сайт обновляется автоматически
- Файл `_headers` содержит заголовки кэширования и безопасности
- Файл `_redirects` обеспечивает чистые URL без `.html`
- Файл `CNAME` указывает на домен `broshio.ru`

Также работает на GitHub Pages, Netlify и любом статическом хостинге.

## Переносимость

Хочешь сменить хостинг или просто перенести сайт на другой компьютер — скопируй всю папку проекта. Никаких миграций баз данных, никаких зависимостей.

| Что перенести | Где лежит |
|--------------|-----------|
| Все работы и фото | `works/` |
| Тексты и контакты | `config.json` |
| GitHub-токен | `secrets.json` |
| Весь сайт | вся папка целиком |
